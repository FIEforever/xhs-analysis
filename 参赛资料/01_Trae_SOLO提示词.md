# 小红书内容分析系统 — Trae SOLO Web版 完整提示词

> **适用版本**：Trae SOLO 浏览器 Web 版（solo.trae.cn）  
> **使用方式**：将"第三章：发给SOLO的完整提示词"整段复制，粘贴到 SOLO 对话框发送  
> **SOLO 模式选择**：选 **SOLO Builder**（构建 Web 应用）

## 一、Trae SOLO Web 版能力说明

| 能力 | Web 版支持情况 |
|------|---------------|
| 生成单文件 HTML | ✅ 完全支持 |
| 在线预览效果 | ✅ 内置浏览器预览 |
| Vercel 部署 | ✅ 一键部署到线上 |
| 访问本地 exe 文件 | ❌ 不支持（云端沙箱） |
| 执行 PowerShell 命令 | ❌ 不支持 |

**重要结论**：SOLO Web 版无法直接调用本机的 `xiaohongshu-mcp-windows-amd64.exe`。  
解决方案：

1. **先让 SOLO 生成完整的分析系统**（含内嵌50条演示数据，可以立即运行预览）
2. **本地启动 MCP 服务后**，在系统设置页填入 `http://localhost:18060` 地址，即可切换到真实数据模式
3. 这样展示的时候：演示模式可在任何地方展示，真实数据模式在本机上展示

## 二、系统架构说明（给 SOLO 参考）

```
┌─────────────────────────────────────────────────┐
│               小红书内容分析平台 (index.html)       │
│                                                 │
│  搜索面板 → [调用 REST API] → 数据归一化 → 图表渲染   │
│               ↓                                 │
│   优先 http://{自定义地址}/api/v1/feeds/search     │
│               ↓ 如连接失败                         │
│   降级到内置演示数据（50条，文创冰箱贴主题）           │
└─────────────────────────────────────────────────┘
```

**MCP REST API 规格**（本地启动后使用）：

```
健康检查：GET http://localhost:18060/health
         响应：{"success":true,"data":{"status":"healthy"}}

搜索接口：GET http://localhost:18060/api/v1/feeds/search
         参数：keyword=文创冰箱贴&sort=general&filter_duration=2
         
filter_duration 映射：
  1 = 最近1天
  2 = 最近1周（7天）
  3 = 最近半年（30天/90天）
```

**真实响应数据格式**（字段全部驼峰命名）：

```json
{
  "success": true,
  "data": {
    "feeds": [
      {
        "id": "67890abcdef123456",
        "xsecToken": "BGPzrHKRxxxxxxxxxx",
        "noteCard": {
          "type": "normal",
          "displayTitle": "超可爱的故宫文创冰箱贴，太适合送朋友了！",
          "user": { "nickname": "爱生活的露西", "userId": "5f3a1b2c3d4e5f6" },
          "interactInfo": { "likedCount": 2856, "collectedCount": 1342, "commentCount": 287 },
          "cover": { "urlDefault": "https://sns-img-qc.xhscdn.com/xxxxxx.jpg" }
        }
      }
    ]
  }
}
```

## 五、本地真实数据连接（发帖后在评论区演示用）

在 SOLO 生成的系统可正常展示演示数据后，连接本地 MCP 获取真实数据：

```powershell
# 步骤1：登录（只需一次）
cd d:/F/NW/小红书
./xiaohongshu-login-windows-amd64.exe
# → 浏览器弹出 → 手机扫码登录小红书 → 自动保存 cookies.json

# 步骤2：启动 MCP 服务（保持窗口开着）
./xiaohongshu-mcp-windows-amd64.exe
# → 显示 "Server started on :18060" 即成功

# 步骤3：验证服务
# 浏览器访问：http://localhost:18060/health
# 应返回：{"success":true,"data":{"status":"healthy"}}
```

## 六、常见问题 Q&A

| 问题 | 解决 |
|------|------|
| SOLO 生成的代码词云不显示 | 追加提示："请确保词云用 Canvas API 实现，数据来自演示数据的标题字段" |
| 图表显示空白 | 追加提示："请检查演示数据是否正确内嵌，Chart.js 图表初始化需在 DOM 渲染后执行" |
| 移动端 Tab 显示不全 | 追加提示："Tab 导航在小屏设备请添加 overflow-x:auto 横向滚动" |
| 点击卡片无法跳转 | 追加提示："笔记卡片的 cursor:pointer + window.open 跳转逻辑请检查" |
| Vercel 部署失败 | 单 HTML 文件直接部署到 Vercel Static，无需 package.json |
