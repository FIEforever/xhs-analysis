# Trae SOLO Web 版 — 操作流程说明

> 本文档专门针对 **solo.trae.cn 浏览器版** 的操作步骤，Windows 客户端版不出来之前按此操作

---

## 核心理解：Web 版 vs 本地版的区别

| 对比项 | SOLO 本地客户端版 | SOLO Web 版（当前可用） |
|--------|-----------------|----------------------|
| 能否运行 exe | ✅ 可以 | ❌ 不可以（云端沙箱） |
| 能否访问 localhost | ✅ 可以 | ❌ 不可以 |
| 生成 HTML | ✅ | ✅ |
| Vercel 在线部署 | ✅ | ✅ |
| 浏览器预览 | ✅ | ✅ |

**Web 版策略**：让 SOLO 生成一个**演示模式完整可用**的系统，MCP 地址做成可配置项，本地运行 MCP 服务后在设置里填入地址切换真实数据。

---

## 完整操作步骤

### 【一】打开 SOLO Web 版

访问：https://solo.trae.cn

- 点击"新建任务"或进入已有 Session
- 模式选择：**SOLO Builder**（不是 Coder，Builder 专门做 Web App）

---

### 【二】发送提示词

打开 `01_Trae_SOLO提示词.md`，找到 **"三、发给 SOLO 的完整提示词"** 章节。

复制三个反引号 ` ``` ` 之间的所有内容（从"请帮我构建"开始，到最后一行结束）。

粘贴到 SOLO 对话框，**不要修改任何内容**，直接发送。

> 💡 提示：如果 SOLO 提示"内容过长"，可以分两次发送：
> - 第一次：发送到"【Tab 2：产品洞察】"前的部分
> - 第二次：发送"继续，接上文：【Tab 2：产品洞察】..."

---

### 【三】确认 SOLO 的规划

SOLO Builder 会先生成一份规划文档（Spec），内容类似：

```
我会为你构建一个小红书内容智能分析平台，包含：
1. 搜索面板（关键词+时间段+数量）
2. 4个分析Tab（运营/产品/消费者/缓存）
3. 笔记列表（4列卡片网格）
4. 演示数据内嵌（50条）
...
```

直接点击"**确认执行**"即可，不需要修改。

---

### 【四】等待生成（约5-10分钟）

SOLO 会自动：
1. 生成 `index.html` 主文件
2. 填充50条演示数据
3. 实现 Chart.js 图表
4. 实现词云（Canvas手写）
5. 在右侧预览面板显示效果

**期间要做的事**：截图！截图！截图！

关键截图时机：
- SOLO 开始写代码时（显示代码生成进度）
- SOLO 在左侧对话框描述每个模块时
- SOLO 完成并显示预览效果时

---

### 【五】检查生成结果

在 SOLO 内置预览中依次检查：
- [ ] 顶部导航栏显示"演示模式"灰色状态徽章
- [ ] 点击"开始分析"按钮，3个红点加载动画出现
- [ ] 运营分析 Tab：柱状图、词云、饼图都显示
- [ ] 产品洞察 Tab：条形图、场景标签显示
- [ ] 消费者洞察 Tab：情感分析圆形进度条、雷达图显示
- [ ] 笔记列表：4列卡片，有封面占位图，数字显示
- [ ] 移动端：缩小浏览器到375px宽，卡片变2列

**如果某个部分显示有问题**，追加提示，例如：
- "词云没有显示，请检查Canvas绘制代码并修复"
- "运营分析的柱状图是空的，请确认Chart.js初始化代码"
- "点击笔记卡片没有跳转，请添加window.open跳转逻辑"

---

### 【六】Vercel 一键部署（强烈建议，评分加分）

1. 在 SOLO 界面找到"部署"按钮（SOLO Builder 右上角或对话框输入"帮我部署到Vercel"）
2. 授权 Vercel 账号（没有的话先去 vercel.com 注册，免费）
3. 部署完成后获得线上地址，如：`https://xhs-analysis-platform-xxx.vercel.app`
4. **把这个地址记录下来**，发帖时放在"在线体验"区域，让评审可以直接点击体验

---

### 【七】本地连接真实数据（可选，本机演示用）

SOLO 已生成系统后，按以下步骤连接真实小红书数据：

**步骤1：确认 cookies.json 有效**

用记事本打开 `d:/F/NW/小红书/cookies.json`，确认有 `web_session` 字段。

**步骤2：启动 MCP 服务**

```powershell
cd d:/F/NW/小红书
./xiaohongshu-mcp-windows-amd64.exe
```

保持窗口开着，看到 `Server started on :18060` 表示成功。

**步骤3：配置系统地址**

在浏览器打开 SOLO 生成的系统 → 点右上角 ⚙️ 设置 → 
填入 `http://localhost:18060` → 点"测试连接" → 
看到绿色"已连接"→ 保存

**步骤4：搜索真实数据**

输入关键词（如"文创冰箱贴"）→ 点"开始分析" → 等待约30秒 → 真实数据出现。
