# 【Code with SOLO】用 Trae SOLO MTC 模式 30 分钟搭建小红书内容智能分析平台

## 项目地址

- **在线体验**：https://FIEforever.github.io/xhs-analysis
- **GitHub 源码**：https://github.com/FIEforever/xhs-analysis

## 技术路径

```
Trae SOLO MTC 模式（执行生成）
    ↓ 粘贴精心设计的提示词
单文件 index.html（30分钟生成）
    ↓
本地 MCP 程序（提供真实数据）
    ↓
浏览器预览
```

## 核心能力

1. **提示词工程 → AI 生成代码**：用 MTC 模式多轮对话，精确控制生成结果
2. **真实数据**：本地运行 MCP 程序，通过 REST API 获取小红书真实笔记数据
3. **双模式设计**：MCP 运行用真实数据，关掉自动降级到演示数据
4. **7 种可视化图表**：词云、热力图、雷达图、柱状图、饼图、折线图等

## 技术亮点（踩过的坑）

| 问题 | 解决方案 |
|------|---------|
| MCP 协议 session bug | 改用 REST API 直接调用 |
| 服务端爬取超时 30 秒 | 前端超时设置为 60 秒 |
| 封面图防盗链 | onerror 显示灰色占位图 |
| 驼峰字段名 | 统一数据归一化处理 |

## 使用方式

### 方式一：在线体验（演示模式）
直接访问：https://FIEforever.github.io/xhs-analysis

### 方式二：真实数据模式
1. 下载 MCP 程序：xiaohongshu-mcp-windows-amd64.exe
2. 本地运行：`./xiaohongshu-mcp-windows-amd64.exe`
3. 打开 http://localhost:18060/health 确认连接成功
4. 在系统中配置 MCP 地址，开始爬取真实数据
