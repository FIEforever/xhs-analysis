## 项目描述（500字）

### 完整描述

```
【项目背景】

作为内容运营从业者，需要频繁分析小红书上的内容热度、用户画像和竞争态势。传统方式靠人工搜索，数据分散、效率低下。

【解决方案】

我使用 Trae SOLO 的 MTC（Multi-Turn Chat）多轮对话模式，通过精心设计的提示词，30 分钟生成了完整的小红书内容智能分析系统。

【技术架构】

- 前端：单文件 HTML，Chart.js 实现 7 种可视化图表
- 数据层：本地 MCP 程序提供 REST API（字节跳动官方数据服务）
- 降级策略：MCP 离线时自动切换到内置 50 条演示数据

【核心功能】

1. 运营分析：互动量分布、最佳发布时间、热门标签词云、内容类型占比
2. 产品洞察：关键词频次、使用场景、产品形态、互动热度
3. 消费者画像：情感分析、需求词/痛点词、互动行为偏好雷达图
4. 数据缓存：localStorage 缓存，24小时有效期

【技术亮点】

- MCP REST API 接入真实小红书数据（非演示数据）
- 60 秒超时处理（服务端爬取需要 20-30 秒）
- 封面图防盗链占位图处理
- 响应式布局，支持移动端

【提效数据】

- 系统生成：从 3-5 天 → 30 分钟
- 数据搜索：从 2-3 小时 → 30 秒
- 图表生成：从手动制作 → 自动渲染

【使用方式】

1. 在线体验：访问 https://FIEforever.github.io/xhs-analysis 即可查看演示模式
2. 真实数据：在本地运行 xiaohongshu-mcp-windows-amd64.exe，配置地址后即可爬取真实数据
```
